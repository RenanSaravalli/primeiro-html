Client x Server

(voltar)

Cliente:

O cliente é como uma criança que quer ler um livro. Essa criança pega um computador ou um tablet, abre um navegador (como o Chrome ou o Firefox) e digita o endereço do site que quer visitar, como se estivesse pedindo um livro específico na biblioteca.

Servidor:

O servidor é como um bibliotecário. Ele é quem guarda todos os livros (que, na web, são as páginas e os arquivos dos sites). Quando a criança (cliente) pede um livro (site), o bibliotecário (servidor) vai buscar esse livro e entrega para a criança.

Inspetor de Elementos

(voltar)

Quando um desenvolvedor quer criar ou arrumar um site, ele usa o inspetor de elementos para ver e mexer em todas as partes da página. Aqui está como funciona: Para usar o inspetor de elementos, o desenvolvedor pode clicar com o botão direito em qualquer lugar da página e selecionar "Inspecionar" ou "Inspecionar Elemento". Também pode usar um atalho no teclado, como F12 ou Ctrl+Shift+I. Assim que o inspetor de elementos é aberto, ele mostra o código da página. Esse código é escrito em HTML (para a estrutura da página), CSS (para a aparência) e JavaScript (para a interatividade). É como ver os "ingredientes" que compõem a página.
O inspetor permite que o desenvolvedor clique em diferentes partes da página e veja o código correspondente. Por exemplo, se clicar em uma imagem, verá o código HTML que mostra como essa imagem está inserida na página. Se algo não está funcionando direito, o inspetor de elementos ajuda a encontrar e corrigir erros. O desenvolvedor pode ver mensagens de erro e descobrir o que está errado no código JavaScript, por exemplo.
O inspetor de elementos é essencial para desenvolvedores web, pois permite que vejam exatamente como a página funciona e façam ajustes precisos para melhorá-la

Estrutura básica do HTML

(voltar)

Imagine que um documento HTML é como uma receita de bolo. Cada parte da receita ajuda a criar o bolo completo, e cada parte do HTML ajuda a criar uma página web. Vamos ver como funciona:

  1. <!DOCTYPE html>
    • O que é: Uma declaração que diz ao navegador que tipo de documento ele está prestes a ler.
    • Analogia: É como o título da receita que diz "Esta é uma receita de bolo de chocolate".
  2. <html>
    • O que é: A raiz do documento HTML. Tudo dentro deste elemento é parte da página web.
    • Analogia:É como a capa do livro de receitas que envolve todas as páginas da receita.
  3. <head>
    • O que é: A raiz do documento HTML. Tudo dentro deste elemento é parte da página web.
    • Analogia:É como a capa do livro de receitas que envolve todas as páginas da receita.
  4. <title>
    • O que é: Define o título da página que aparece na aba do navegador.
    • Analogia:É como o nome da receita, "Bolo de Chocolate", que aparece no topo da página da receita.
  5. <body>
    • O que é: Contém o conteúdo visível da página, como textos, imagens, vídeos, etc.
    • Analogia:É como a parte da receita que contém as instruções passo a passo para fazer o bolo.
  6. <h1>
    • O que é: Um título de nível 1, geralmente usado para o título principal da página.
    • Analogia:É como o título de uma seção importante na receita, como "Ingredientes".
  7. <p>
    • O que é: Define um parágrafo de texto.
    • Analogia:É como uma linha de instrução na receita, como "Misture a farinha com o açúcar".

Listas ordenadas e não ordenadas

(voltar)

Listas em HTML são usadas para agrupar itens relacionados de uma forma organizada. Existem dois tipos principais de listas: listas ordenadas e listas não ordenadas. Vamos explorar cada uma delas.

Listas Não Ordenadas (Unordered Lists)

As listas não ordenadas são usadas quando a ordem dos itens não é importante. Cada item na lista é precedido por um marcador, geralmente um ponto.

Explicação:

  • <ul> Significa "unordered list" (lista não ordenada). Envolve toda a lista.
  • <li> Significa "list item" (item da lista). Cada item da lista é envolvido por essa tag.

Listas Ordenadas (Ordered Lists)

As listas ordenadas são usadas quando a ordem dos itens é importante. Cada item na lista é precedido por um número.

Explicação:

  • <ol> Significa "ordered list" (lista ordenada). Envolve toda a lista.
  • <li> Cada item da lista é envolvido por essa tag.

Listas de Definição (Definition Lists)

Além das listas ordenadas e não ordenadas, existe também a lista de definição, usada para pares de termos e suas definições.

Explicação:

  • <dl> Significa "definition list" (lista de definição). Envolve toda a lista.
  • <dt> Significa "definition term" (termo de definição). Representa o termo a ser definido.
  • <dd> Significa "definition description" (descrição de definição). Representa a definição ou descrição do termo.

Link

Um link em HTML, também conhecido como "hiperlink", é um elemento que permite a navegação entre páginas da web ou recursos dentro da mesma página. Ele é criado usando a tag , que significa "anchor" (âncora). Vamos ver como funciona.

<a href="URL">Texto do Link</a>

Explicação:

  • <a> Tag que define o início do link.
  • <href> Atributo que especifica o destino do link, ou seja, a URL para onde o link aponta.

  • Texto do Link:

    O texto que será clicável. Pode ser uma palavra, frase ou até mesmo uma imagem.
